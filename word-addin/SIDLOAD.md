# Установка надстройки Legal AI для Microsoft Word

## Вариант 1: Разработка на одном ПК (localhost)

Подходит для разработки и тестирования на вашем Windows-ПК.

### Предварительные требования
- Docker Desktop запущен, все сервисы работают (`docker compose up -d`)
- Бэкенд доступен на `http://localhost:8000`
- Microsoft Word 2016+ (Windows)

> ⚠️ **Только Windows.** На Mac Office требует HTTPS даже для localhost.

### Шаг 1. Проверьте, что бэкенд раздаёт надстройку

Откройте в браузере: http://localhost:8000/addin/taskpane.html
Должна появиться панель Legal AI Assistant.

### Шаг 2. Загрузите надстройку (Sideload)

**Способ A — через сетевую папку (рекомендуется):**

1. Откройте Word → **Файл** → **Параметры** → **Центр управления безопасностью** → **Параметры центра управления безопасностью…**
2. Слева выберите **Надстройки, не входящие в каталог надстроек** (или **Trusted Add-in Catalogs**).
3. В поле **URL-адрес каталога** введите путь к папке с манифестом:
   ```
   \\ваш-ПК\shared\legal-ai\
   ```
   или локальный путь:
   ```
   C:\LegalAI\word-addin\
   ```
4. Нажмите **Добавить каталог** → убедитесь, что он появился в списке.
5. Поставьте галочку «Показывать в меню» → ОК → перезапустите Word.

**Способ B — загрузка вручную (для быстрого теста):**

1. Откройте Word.
2. Вкладка **Вставка** → **Мои надстройки** → **Загрузить мою надстройку** (Upload My Add-in).
3. Выберите файл `manifest.xml` из папки `word-addin/`.

### Шаг 3. Запустите надстройку

1. В Word откройте (или создайте) документ.
2. Вкладка **Вставка** → **Мои надстройки** → выберите **Legal AI Assistant**.
3. Справа откроется панель надстройки.
4. Нажмите **«🔍 Анализировать документ»**.

### Что произойдёт
1. Надстройка получит текст документа через Office.js
2. Отправит в бэкенд `POST /api/analyze/text`
3. Покажет отчёт в панели (резюме, риски, рекомендации, нормы)
4. Подсветит проблемные места прямо в тексте Word:
   - 🔴 красным — высокие риски
   - 🟡 жёлтым — средние риски / рекомендации
   - 🟢 зелёным — низкие риски

Клик по риску или рекомендации в панели переносит курсор к этому месту в документе. Клик по норме права открывает полный текст в модальном окне.

## Вариант 2: Локальная сеть (несколько ПК)

Для использования надстройки на нескольких ПК в локальной сети.

### Шаг 1. Разверните бэкенд на сервере

Разверните проект на сервере в сети (через `docker compose up -d`).
Бэкенд будет доступен по адресу `http://IP-сервера:8000`.

### Шаг 2. Настройте HTTPS (обязательно для сети!)

Word по умолчанию блокирует HTTP-источники (кроме localhost). Для сети нужен HTTPS.

**Простейший вариант — self-signed сертификат через nginx:**

Создайте `nginx.conf`:
```
server {
    listen 443 ssl;
    server_name ваш-сервер;
    ssl_certificate     /etc/nginx/cert.pem;
    ssl_certificate_key /etc/nginx/cert.key;

    location / {
        proxy_pass http://backend:8000;
    }
}
```

Сгенерируйте сертификат:
```bash
openssl req -x509 -newkey rsa:4096 -keyout cert.key -out cert.pem -days 365 -nodes
```

### Шаг 3. Обновите манифест

В `manifest.xml` замените все `http://localhost:8000` на ваш HTTPS-адрес:
```
https://ваш-сервер/addin/taskpane.html
```

Также в `taskpane.js` замените `API_BASE`:
```js
const API_BASE = "https://ваш-сервер";
```

### Шаг 4. Распространите манифест

Положите `manifest.xml` на сетевой ресурс:
```
\\server\share\legal-ai-manifest.xml
```

На каждом клиентском ПК:
1. Word → Файл → Параметры → Центр управления безопасностью
2. Добавьте путь `\\server\share\` в доверенные каталоги надстроек
3. Перезапустите Word → Вставка → Мои надстройки → Legal AI

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| Надстройка не появляется | Проверьте, что путь к манифесту добавлен в доверенные каталоги, перезапустите Word |
| Панель пустая / ошибка загрузки | Убедитесь, что бэкенд работает: откройте `http://localhost:8000/addin/taskpane.html` в браузере |
| «Не удалось получить текст» | Откройте или создайте документ Word, затем нажмите «Анализировать» |
| Подсветка не работает | Нажмите «Анализировать» заново; проверьте, что у надстройки есть права на редактирование |
| Ошибка сети на клиентском ПК | Проверьте доступность бэкенда с этого ПК (`ping IP-сервера`, откройте URL в браузере) |
| Office.js не грузится | Нужен интернет для первого открытия (CDN). Альтернатива — захостить `office.js` локально |

## Ограничения
- **Windows only** для localhost HTTP. На Mac требуется HTTPS.
- **Размер документа** — Office.js `getBodyAsync` имеет лимит ~1МБ текста. Для очень больших документов может потребоваться chunking.
- **Подсветка через search** — если одинаковая цитата встречается несколько раз, подсветятся все вхождения.
- **Офлайн** — при первом открытии нужен интернет (Office.js грузится с CDN Microsoft).
