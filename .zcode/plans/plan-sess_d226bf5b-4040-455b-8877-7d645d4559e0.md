# План: надстройка Microsoft Word (Office Web Add-in)

## Архитектура

Надстройка = тонкий веб-клиент (HTML+JS+Office.js) в боковой панели Word, который берёт текст документа через Office.js API и отправляет в наш FastAPI бэкенд (GigaChat + Qdrant + RAG уже готовы).

```
Word (Win)                          FastAPI бэкенд
┌─────────────────────┐             ┌────────────────────┐
│ Документ Word       │             │ POST /api/analyze/ │
│   ↑ Office.js       │  1.текст    │      text (НОВЫЙ)  │
│ Task pane (iframe)  │────────────►│                    │
│   taskpane.html     │             │ RAG + GigaChat     │
│   taskpane.js       │◄────────────│ + Qdrant (готово)  │
│   ↑ Office.js       │  2.результат│                    │
│ ↓ подсветка range   │             │ StaticFiles раздаёт│
└─────────────────────┘             │ taskpane.html (НОВ)│
                                    └────────────────────┘
```

## Что меняется в бэкенде

### 1. Новый эндпоинт `POST /api/analyze/text` (analysis.py)
Принимает JSON `{ "text": "...", "file_name": "document.docx" }`, а не файл. Переиспользует существующий pipeline:
- `analyze_document()` рефакторится: внутреннюю часть (embedding → search → LLM) вынести в `_run_analysis(text, file_name)`, чтобы и файловый, и текстовый эндпоинты делили логику
- Текстовый эндпоинт пропускает `parse_document` (текст уже есть от Word)
- Возвращает тот же `AnalysisResponse`

### 2. StaticFiles в main.py
```python
from fastapi.staticfiles import StaticFiles
app.mount("/addin", StaticFiles(directory="word-addin", html=True), name="addin")
```
Надстройка доступна на `http://localhost:8000/addin/taskpane.html`

### 3. CORS — добавить домены Office
В `.env` расширить `CORS_ORIGINS`:
```
CORS_ORIGINS=http://localhost:5173,https://localhost:3000,https://localhost:8000,null
```
(Office иногда шлёт запросы с origin `null` из iframe — добавляем).

## Новые файлы надстройки (`word-addin/`)

### `manifest.xml`
Официальный XML-манифест Office Add-in:
- `<ProviderName>`, `<DisplayName>`, `<Version>`
- `<Hosts>` = `Workbook`/`Document` (Word)
- `<DefaultSettings>` → ` <SourceLocation>` указывает на `http://localhost:8000/addin/taskpane.html` (для разработки)
- `<Permissions>` = `ReadWriteDocument` (для подсветки)
- Точка расширения `PrimaryCommandSurface` — кнопка «Legal AI» на ленте

### `taskpane.html`
Боковая панель (минималистичный UI):
- Кнопка «🔍 Анализировать документ»
- Область результата: резюме, список рисков (кликабельные), рекомендации, нормы
- Индикатор загрузки
- Связано стилями с основным проектом (тёмная тема)

### `taskpane.js`
Логика на чистом JS с Office.js:
```
1. Office.onReady() → инициализация
2. Кнопка «Анализировать»:
   - Office.context.document.body.getTextAsync() → получить весь текст
   - POST /api/analyze/text { text, file_name }
   - Отрисовать отчёт в панели
3. Подсветка в документе:
   - Для каждого риска/рекомендации с quote:
     body.search(quote) → range.items[0]
     range.format.highlightColor = цвет по severity
     range.insertComment(описание) (опционально)
4. Клик по риску в панели → range.select() (перенос курсора)
5. Клик по норме → Dialog API открывает полный текст
```

**Цвета подсветки в Word** (Hex-коды, согласованы с веб-интерфейсом):
- Высокий риск: `#F87171` (красный)
- Средний риск/рекомендация: `#FBBF24` (жёлтый)
- Низкий риск/соответствие: `#34D399` (зелёный)

### `commands.html` (пустая страница для командного фрейма)
Стандартный файл, который требует Office для регистрации команд надстройки.

## Этапы реализации

### Этап 1 — Бэкенд: текстовый эндпоинт + StaticFiles
- Рефакторинг `analyze_document`: вынести `_run_analysis(text, file_name)`
- Новый эндпоинт `POST /api/analyze/text` (JSON-вход)
- `main.py`: `StaticFiles` для `/addin`
- CORS: расширить origins

### Этап 2 — Манифест надстройки
- `manifest.xml` со ссылкой на `localhost:8000/addin/taskpane.html`
- Описать кнопку на ленте, разрешения ReadWriteDocument

### Этап 3 — Веб-часть (taskpane)
- `taskpane.html` — разметка панели
- `taskpane.js` — Office.js логика: получить текст → POST → отрисовать отчёт
- `taskpane.css` — стили (встроены или отдельным файлом)
- `commands.html` — пустая заглушка

### Этап 4 — Подсветка в Word (расширенный функционал)
- Поиск каждой цитаты через `body.search()`
- Установка `highlightColor` по severity
- Очистка подсветки перед повторным анализом

### Этап 5 — Инструкция по установке (SIDLOAD.md)
Пошаговая инструкция для Windows:
1. Положить `manifest.xml` на сетевой диск `\\server\share\manifest.xml`
2. В Word: Файл → Параметры → Центр управления безопасностью → Параметры центра → Надстройки → URL доверенных каталогов → добавить путь
3. Вкладка «Вставка» → «Мои надстройки» → выбрать Legal AI
4. Для разработки на одном ПК: «Загрузить надстройку» с локальным манифестом

## Что НЕ входит в эту итерацию
- HTTPS-сертификат для сетевого развёртывания (сначала localhost)
- Публикация в AppSource (Microsoft Store) — это отдельный процесс сертификации
- Real-time обновление при печати (анализ только по кнопке)
- Поддержка Mac (требует HTTPS даже для localhost)
- Автоматическое определение изменений документа

## Технические ограничения (чтобы знать заранее)
1. **Только Windows для localhost HTTP** — на Mac Office требует HTTPS даже для localhost
2. **Office.js грузится с CDN** — нужен интернет при первом открытии (или можно захостить локально)
3. **Подсветка через search** — если одинаковая цитата встречается несколько раз, подсветятся все вхождения
4. **Размер документа** — Office.js `getTextAsync` имеет лимит ~1МБ текста за вызов; для больших документов нужен chunking
